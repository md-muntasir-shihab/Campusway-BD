============================================================
1. SYSTEM OVERVIEW
============================================================
[Observed] CampusWay is a multi-surface education platform with:
- Public site (universities, news, resources, plans, contact, help)
- Student portal (dashboard, profile, exams, results, payments, support)
- Admin control plane (content, academics, operations, finance, security)

[Observed] Active runtime apps:
- backend (Express + TypeScript + MongoDB)
- frontend (Vite + React primary app)
- frontend-next (hybrid partial surface)

[Observed] Backend entry and route wiring are in backend/src/server.ts.
[Observed] Primary route and UI composition are in frontend/src/App.tsx.

[Inferred] System is feature-rich but has migration debt:
- legacy redirects
- duplicate/legacy model files
- dual permission pathways (legacy and v2)

- support_agent: support-focused scopes
- finance_agent: payments/finance/reports-focused scopes
- chairman: oversight/analytics limited scope
- student: student-only protected routes and self-service APIs
- guest: public read and selected anonymous writes (protected by app-check/rate limits where wired)

Permission mismatch risks
